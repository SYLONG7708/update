申悅車機助手更新中心零基礎教學

這份教學給完全不懂程式的人使用。新版不是獨立新增「申悅車機助手」下載卡片，而是把完整 APK 更新中心整合到申悅車機助手的「更新」分頁裡。

你現在已經有的檔案

  • 新版助手 APK：releases\shen-yue-car-assistant-update-center-v2.0.6.apk
  • 雲端頁面資料夾：cloud-upload
  • 雲端更新清單：cloud-upload\updates.json
  • 每個 App 的小圖標：cloud-upload\assets\update-icons
  • 每個 App 的詳情圖片：cloud-upload\assets\update-gallery
  • APK 來源資料夾：C:\Users\Administrator\Downloads\更新

新版更新中心位置

  1. 打開 申悅車機助手。
  2. 點首頁或下方導覽列的 更新。
  3. 畫面會進入完整 APK 更新中心。
  4. 每個 APK 會顯示一個可點擊圖標。
  5. 點圖標後會進入單獨頁面，顯示小圖標、介紹、版本、大小、兩張圖片位置與下載按鈕。

申悅車機助手本身不會出現在 APK 更新清單內；它是主程式入口。首頁導覽列目前只保留「首頁、服務、教學、保固、更新、聯絡」，已移除「交車」。

重要限制：Android 仍會要求確認安裝

一般 APK 沒有系統權限，不能偷偷靜默安裝或替換其他 App。這個更新中心可以讓雲端清單、圖片、介紹、APK 下載來源隨時更新；實際安裝時，Android 系統仍會跳出確認畫面。

如果車機廠商提供系統權限或靜默安裝權限，才可能做到完全免確認安裝。

第一步：放入要上架的 APK

  1. 打開資料夾：C:\Users\Administrator\Downloads\更新
  2. 把要提供下載更新的 APK 放進去。
  3. 助手.apk 會自動排除，不會顯示在更新中心。
  4. 如果檔名是 卡1.apk，更新中心會顯示為 卡1。
  5. 同一個 App 要更新時，新 APK 的 versionCode 必須比舊版高。

第二步：產生更新清單、圖標與圖片位置

在專案資料夾執行：

powershell -ExecutionPolicy Bypass -File .\tools\import-update-apks.ps1

它會自動完成：

  • 解析每個 APK 的名稱、版本、大小與下載檔。
  • 複製 APK 到 cloud-upload\apks。
  • 產生 App 小圖標到 cloud-upload\assets\update-icons。
  • 每個 App 產生兩張圖片位置到 cloud-upload\assets\update-gallery。
  • 更新 cloud-upload\updates.json。

第三步：重新建置申悅車機助手 APK

powershell -ExecutionPolicy Bypass -File .\tools\build-release.ps1

建置完成後，APK 會在：

releases\shen-yue-car-assistant-update-center-v2.0.6.apk

第四步：一鍵上傳 GitHub

你已安裝 GitHub CLI，位置是：

D:\Program Files\GitHub CLI\gh.exe

執行：

powershell -ExecutionPolicy Bypass -File .\tools\upload-github-cloud.ps1

它會自動上傳到：

  • https://github.com/SYLONG7708/shen-yue-iphone-assistant：申悅車機助手網頁與更新中心頁面。
  • https://github.com/SYLONG7708/update：更新中心清單、圖標、圖片。
  • https://github.com/SYLONG7708/update/releases/tag/apk-cloud：大型 APK 檔案。
  • https://github.com/SYLONG7708/shen-yue-iphone-assistant/releases/tag/assistant-apk：新版申悅車機助手 APK。

第五步：車機端使用

  1. 把 shen-yue-car-assistant-update-center-v2.0.6.apk 安裝到安卓車機。
  2. 打開 申悅車機助手。
  3. 點 更新。
  4. 第一次使用先按 安裝權限，允許未知來源安裝。
  5. 按 重新整理。
  6. 點 App 項目進入單獨頁面。
  7. 確認介紹、版本與圖片後按 下載安裝。

如何替換單獨頁面的兩張圖片

  1. 打開 cloud-upload\updates.json。
  2. 找到要修改的 App。
  3. 找到 galleryImages，裡面會有兩個圖片檔名。
  4. 到 cloud-upload\assets\update-gallery。
  5. 用自己的 PNG 或 JPG 圖片覆蓋同名檔案。
  6. 檔名保持一樣，重新執行上傳腳本即可。

如何修改介紹文字

打開 cloud-upload\updates.json，找到對應 App：

欄位用途
name圖標下方與單獨頁標題。
description單獨頁介紹文字，可長按複製。
iconUrl更新中心小圖標。
galleryImages單獨頁兩張圖片位置。
versionName畫面顯示版本。
apkUrl下載安裝時使用，畫面不顯示給客戶。
sha256下載驗證使用，畫面不顯示給客戶。

大檔 APK 注意事項

GitHub 一般倉庫不適合直接放超過 100 MB 的大 APK。這版已改用 GitHub Releases 放 APK，所以 Netflix、卡1、無廣告等大檔可以正常放到 Release 下載。

常見問題

打開更新頁沒有看到檔案

先確認 updates.json 已上傳到 SYLONG7708/update，再按「重新整理」。如果 GitHub 暫時讀不到，APK 內建清單會作為備援。

按下載後顯示「已是最新」

代表車機內已安裝版本碼大於或等於雲端版本碼。請確認新版 APK 的 versionCode 有變高。

下載後不能覆蓋安裝

通常是包名相同但簽章不同。Android 會要求先移除舊版再安裝。

我不想顯示下載網址、包名、SHA

新版畫面已隱藏這些欄位，只保留系統下載與驗證使用。
